# 验收标准

## A. 基础运行

1. 项目在 Node.js 20 环境下可以安装依赖并启动。
2. 执行 `npm run dev` 后，本地页面可访问。
3. 执行 `npm run lint`、`npm run typecheck`、`npm run test` 能通过。

## B. 地图能力

1. 页面首页默认显示上海地图。
2. 用户可以缩放、拖拽地图。
3. 地图有合法 attribution。
4. 住宅图层默认显示。

## C. 住宅展示

1. 低缩放级别下，住宅以聚合形式展示。
2. 高缩放级别下，住宅可展开为单点。
3. 聚合数量与展开逻辑正确。
4. 住宅点位有选中反馈。

## D. 行政区统计

1. 可显示上海行政区边界或统计层。
2. 至少支持显示各区住宅数量摘要。
3. 数量统计与已加载数据一致。

## E. 住宅详情

1. 点击单住宅后打开详情区域。
2. 详情至少显示：名称、行政区、坐标。
3. 如有数据，显示学区信息。
4. 如有数据，显示交通、医院、商超信息。

## F. 搜索

1. 支持按住宅名或地址关键词搜索。
2. 点击搜索结果后地图定位到目标。
3. 定位后可打开该住宅详情。

## G. 学区信息

1. 支持结构化学区数据导入。
2. 学区信息在详情页可展示。
3. 若线上抓取未启用，不影响主应用运行。

## H. 自动化测试

至少通过以下 E2E 场景：

1. 首页地图成功渲染。
2. 点击住宅后详情抽屉打开。
3. 搜索目标住宅后地图定位成功。

## I. 可替换性

1. 地图数据源可替换。
2. 学区 provider 可替换。
3. nearby provider 可替换。
4. mock 数据与真实数据可通过配置切换。
