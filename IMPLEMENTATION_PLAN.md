# 分阶段开发计划

## Phase 0 - 初始化

目标：建立可运行工程与基础工具链。

任务：

- 初始化 Next.js + TypeScript + Tailwind
- 安装 MapLibre GL、Turf、Zod、Playwright、Vitest
- 配置 ESLint、Prettier、Husky（可选）
- 配置环境变量读取
- 准备 mock 数据目录

完成标准：

- 本地项目可启动
- 首页可正常渲染基础框架

## Phase 1 - 地图 MVP

目标：完成上海地图浏览体验。

任务：

- 地图组件封装
- 默认定位上海
- 支持拖拽、缩放、地图 attribution
- 加载住宅 GeoJSON 图层
- 用 cluster 展示住宅聚合点
- 用未聚合点展示单住宅

完成标准：

- 用户可以流畅浏览地图
- 可以看到住宅聚合与展开

## Phase 2 - 住宅详情

目标：完成点击住宅查看详细信息。

任务：

- 点击点位获取住宅 ID
- 侧边栏/抽屉组件实现
- 详情 API 与数据联动
- 展示基础字段

完成标准：

- 点击单住宅后能看到详情面板

## Phase 3 - 行政区统计

目标：实现全市级别统计可视化。

任务：

- 加载上海行政区边界数据
- 统计各区住宅数量
- 在低 zoom 级别显示区级统计
- 支持 hover/click 查看区级摘要

完成标准：

- 用户能查看各区住宅数量概览

## Phase 4 - 周边公共资源

目标：展示交通、医院、商超等资源。

任务：

- 确定 POI 数据格式
- 实现 nearby provider
- 详情页中按分类展示 nearby 列表
- 增加距离排序

完成标准：

- 有数据时可展示至少三类 nearby 资源

## Phase 5 - 学区信息

目标：接入学区信息。

任务：

- 定义 school-district schema
- 实现 file-based provider
- 实现 school-district importer 脚本
- 若合规且稳定，可添加独立爬虫模块

完成标准：

- 至少可通过本地 JSON/CSV 导入并展示学区信息

## Phase 6 - 搜索与联动

目标：提升可用性。

任务：

- 搜索输入框
- 搜索结果列表
- 点击搜索结果后地图飞行定位
- 自动打开详情面板

完成标准：

- 用户可按名称或地址关键词定位住宅

## Phase 7 - 测试与优化

目标：保证可交付性。

任务：

- 单元测试
- E2E 测试
- 错误态、空态、加载态
- 性能优化（图层刷新、请求节流、防抖）

完成标准：

- 关键测试通过
- 应用具备基本稳定性
