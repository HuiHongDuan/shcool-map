# 上海住房信息地图 Web UI - 产品需求文档

## 1. 项目背景

目标是构建一个围绕上海民用住宅分布与配套信息的地图型 Web 应用。用户可以像使用通用地图导航软件一样浏览地图，并在地图上查看住宅位置、数量、聚合情况，以及每个住宅对应的学区和周边公共资源信息。

## 2. 目标用户

- 上海购房/租房用户
- 家庭置业用户（尤其关注学区）
- 对住宅周边公共资源有强需求的用户
- 需要进行区域对比的研究/咨询人员

## 3. 核心目标

### 3.1 地图浏览

应用必须支持：

- 地图加载
- 放大
- 缩小
- 拖拽平移
- 适配桌面端优先，同时兼容移动端基本查看

### 3.2 住宅可视化

地图上需要高亮显示“民用住宅”信息。

展示要求：

- 在低缩放层级显示聚合点或按区统计
- 在中等缩放层级显示更细粒度聚合
- 在高缩放层级显示单个住宅点位
- 用户能直观看到住宅数量和分布密度

### 3.3 住宅详情

用户点击某个住宅后，应用应展示：

- 住宅名称
- 地址（如有）
- 所属行政区
- 坐标
- 学区信息
- 周边公共资源信息

### 3.4 周边公共资源

至少覆盖以下类别：

- 交通：地铁站、公交站、主要道路/交通节点（可先做前两项）
- 医院：综合医院、社区医院（按数据可得性）
- 商超：商场、超市、便利店（按数据可得性）

每个类别如有数据，应列举：

- 名称
- 类别
- 距离（优先）
- 地址（如有）

### 3.5 学区信息

学区信息需支持查找和展示。来源要求：

- 可以基于公开可访问页面进行采集
- 如果目标网站不适合直接抓取，应支持人工整理后导入
- 数据模型要允许后续替换成更正式的数据源

## 4. 功能需求

## 4.1 地图首页

### 输入

- 用户打开首页

### 输出

- 显示上海范围地图
- 初始视角聚焦上海
- 默认加载住宅分布图层

## 4.2 缩放与聚合

### 输入

- 用户放大或缩小地图

### 输出

- 住宅点位根据缩放级别自动聚合或展开
- 界面不卡顿
- 数量标签准确

## 4.3 选中住宅

### 输入

- 用户点击住宅点位或聚合展开后的单点

### 输出

- 打开详情面板
- 展示住宅相关基础信息
- 展示学区信息（如有）
- 展示周边资源（如有）

## 4.4 搜索

### 输入

- 用户输入住宅名、地址关键词

### 输出

- 返回匹配住宅列表
- 点击结果后地图定位到目标住宅

## 4.5 行政区统计

### 输入

- 用户查看全市级别地图

### 输出

- 可以看到各行政区住宅数量概览
- 鼠标悬停或点击后可显示统计信息

## 5. 非功能需求

### 5.1 性能

- 首屏加载时间尽量控制在可接受范围
- 地图缩放/拖拽应流畅
- 聚合计算应避免明显卡顿

### 5.2 可维护性

- 数据源与 UI 解耦
- 学区数据接入方式可插拔
- 可在 mock 数据与真实数据之间切换

### 5.3 可测试性

- 地图、详情面板、搜索、API 必须可测试
- 关键流程应有自动化测试

### 5.4 合规性

- 地图与数据必须优先采用公开免费资源
- 不以内嵌违规抓取逻辑为前提

## 6. 建议 UI 结构

- 顶部：搜索栏、图层开关、统计摘要
- 左侧或右侧：住宅详情抽屉
- 主区域：地图
- 地图图例：住宅点、聚合点、公共资源分类说明

## 7. 成功标准

满足以下条件则认为首版成功：

1. 用户可用地图查看上海住宅分布
2. 用户可通过缩放感知不同区域住宅数量
3. 用户点击住宅后能看到完整信息面板
4. 学区信息与周边资源可扩展接入
5. 应用具备基本测试与部署能力
